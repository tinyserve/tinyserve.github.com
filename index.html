TinyServe

A local host manager for running small Docker services on your Mac mini. Optional Cloudflare Tunnel integration, and predictable costs with full ownership if you don't need infinite scale or 100% uptime.

Install
$ brew install tinyserve

Getting Started (Mac mini + first deploy)

This is the ideal end-to-end flow: prepare a clean Mac mini, install Docker + tinyserve, then initialize Cloudflare Tunnel and deploy your first service.

1

Base system & remote access (do this first)

Update macOS, install Xcode Command Line Tools (xcode-select --install), create a dedicated admin user, and enable Remote Login (SSH). Optionally enable Screen Sharing for rescue.

Note your machine's IP address and hostname:

Terminal
# Local hostname (usable as <hostname>.local on the same network) scutil --get LocalHostName # IP address (Ethernet) ipconfig getifaddr en0 # IP address (Wi-Fi, if applicable) ipconfig getifaddr en1 # List all network interfaces with IPs ifconfig | grep "inet "

Network recommendations:

  • Set a DHCP reservation or static IP on your router so the Mac mini stays stable.
  • Leave macOS Application Firewall on; tinyserve binds to 127.0.0.1 only.
  • Verify SSH works on your LAN: ssh <user>@<mac-mini-ip> or ssh <user>@<hostname>.local.
  • For off-site access, rely on Cloudflare Tunnel + Access rather than opening router ports.
2

Prevent sleep (critical for headless servers)

Mac mini will sleep when SSH sessions end, causing Cloudflare Tunnel and services to become unreachable. Disable sleep:

Terminal
sudo pmset -a sleep 0 sudo pmset -a disksleep 0 sudo pmset -a displaysleep 0 sudo pmset -a powernap 0 # Verify settings pmset -g

Alternatively via System Settings: Energy - set "Turn display off after" to "Never" and disable automatic sleeping.

3

Install Homebrew

Install Homebrew and add it to your PATH.

Terminal
# Install Homebrew /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # Apple Silicon: add to your shell profile eval "$(/opt/homebrew/bin/brew shellenv)" # Verify brew --version
4

Install a Docker runtime (no Docker Desktop)

Pick one:

  • Colima (recommended): lightweight, fast, and simple.
  • Rancher Desktop with Docker API enabled (GUI option).
  • (Advanced) Lima with Docker socket/nerdctl.
Terminal
# Colima (recommended) brew install colima docker colima start --arch aarch64 --vm-type vz --cpu 4 --memory 8 # Ensure Docker daemon is running docker info docker ps

Optional: auto-start Colima on login with brew services start colima.

Troubleshooting: "Cannot connect to the Docker daemon" means the VM is not running; "docker: command not found" means the CLI is missing; "permission denied" means your user cannot access the socket.

Run tinyserve checklist to verify Docker and other prerequisites.

5

Install tinyserve (Homebrew)

Install tinyserve via Homebrew (core or tap) and verify the CLI.

Terminal
# If tinyserve is in Homebrew core brew install tinyserve # If using a tap brew tap tinyserve/tinyserve brew install tinyserve # Verify tinyserve --help
6

Tinyserve data root

Create the data root (optional, tinyserve will create as needed).

Terminal
mkdir -p "~/Library/Application Support/tinyserve"/{generated,backups,logs,services,traefik,cloudflared}
7

Launch the daemon

Run a foreground sanity check, then install the launchd agent.

Terminal
# Foreground sanity check TINYSERVE_API=http://127.0.0.1:7070 tinyserved # In another terminal tinyserve status # Install as LaunchAgent tinyserve launchd install tinyserve launchd status

Status should return JSON with status: ok during the sanity check.

8

Provide Cloudflare access (recommended)

Prepare a Cloudflare API token with Tunnel + DNS permissions for your domain, then run the one-shot init.

Terminal
tinyserve init \ --domain example.com \ --cloudflare-api-token $CF_API_TOKEN \ --tunnel-name tinyserve-home

tinyserve will create a Cloudflare Tunnel (or reuse one), store credentials under ~/Library/Application Support/tinyserve/cloudflared/, and wire the tunnel to Traefik.

Optional: direct public exposure (no Cloudflare Tunnel) requires:

  • DNS A/AAAA records pointed at your public IP.
  • Router port forwarding for 80/443.
  • Inbound 80/443 allowed on your firewall.
  • TLS termination via a reverse proxy (Traefik, Caddy, Nginx).
  • Dynamic DNS if your public IP changes.
9

Add a service

Add your first service. If --hostname is omitted, tinyserve uses the default domain.

Terminal
tinyserve service add \ --name whoami \ --image traefik/whoami:v1.10 \ --port 80
10

Deploy

tinyserve stages configs, runs docker compose up -d, waits for health, and promotes on success.

Terminal
tinyserve deploy
11

Verify

Check status and logs, then load the URL in your browser.

Terminal
tinyserve status tinyserve logs --service whoami --tail 100

Browser: https://whoami.example.com (optionally behind Cloudflare Access).

12

Backups (recommended)

Set up S3-compatible backups before running production workloads. See docs/BACKUP_RESTORE.md for scripts, schedules, and restore steps.

13

Keep it running

Install/update the launchd agent and keep tinyserve current.

Terminal
# Install/update the launchd agent tinyserve launchd install tinyserve launchd status # Future updates brew upgrade tinyserve && tinyserve launchd uninstall && tinyserve launchd install
